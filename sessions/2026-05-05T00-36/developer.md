## 1. 시스템 아키텍처 설계: 데이터 파이프라인 구조

JAY CORP 시스템은 **'시장 분석 → 운영 최적화 → 수익화 전략'**의 흐름을 자동화하는 것을 목표로 합니다. 따라서 데이터 파이프라인은 **중앙 집중식 데이터 허브(Data Hub)**를 중심으로 각 에이전트가 독립적으로 작업하되, 결과는 상호 연관되도록 설계합니다.

### 1.1. 아키텍처 개요 (Conceptual Flow)

| 단계 | 구성 요소 | 역할 | 주요 데이터 흐름 |
| :--- | :--- | :--- | :--- |
| **Input Layer (수집)** | 외부 API, 웹 크롤러, 내부 DB | 시장 데이터, SCM 데이터, 재무 데이터 등 원천 데이터 수집. | Raw Data $\rightarrow$ Data Ingestion |
| **Processing Layer (처리)** | 분석 에이전트 (Agent 1, 3, 4 등) | 수집된 데이터를 정제, 분석, 예측, 패턴 식별. | Raw Data $\rightarrow$ Processed Data |
| **Decision Layer (결정)** | 전략 에이전트, 재무 에이전트 | 처리된 데이터를 기반으로 최적의 전략(가격, 수량, 마케팅) 결정. | Processed Data $\rightarrow$ Strategy Output |
| **Output Layer (산출)** | Secretary, 최종 보고서 생성 모듈 | 분석 결과, 실행 가능한 스크립트, 보고서 형태로 최종 결과 제공. | Strategy Output $\rightarrow$ Final Report/Action |

### 1.2. 데이터 흐름 상세 (Data Flow Detail)

1.  **Data Ingestion (수집):**
    *   **소스:** 외부 시장 데이터 (크롤링), 내부 운영 데이터 (SCM), 재무 데이터 (매출/비용).
    *   **통합:** 모든 데이터는 **Central Data Store (CDS)**에 표준화된 JSON/CSV 형태로 저장됩니다.

2.  **Transformation & Analysis (변환 및 분석):**
    *   각 에이전트는 CDS에서 필요한 데이터 셋을 추출하여 자체적인 분석 로직(AI/ML)을 적용합니다.
    *   예: Agent 1 (크롤링) $\rightarrow$ Agent 4 (트렌드 분석) $\rightarrow$ Agent 3 (수량 예측).

3.  **Feedback Loop (피드백 루프):**
    *   **핵심:** 분석 결과는 반드시 **재무/KPI 에이전트**로 피드백되어, 모든 전략적 결정이 연매출 목표(100억)에 미치는 영향을 실시간으로 검증합니다.
    *   **결과:** 전략 에이전트가 제시한 결과는 다시 운영 에이전트의 실행 계획으로 연결되어, 실제 시스템에 적용됩니다.

### 1.3. API 및 연동 전략

*   **표준화된 인터페이스:** 모든 에이전트는 내부적으로 **JSON 기반의 표준화된 입출력 포맷**을 사용합니다. 이는 에이전트 간의 데이터 교환 오류를 최소화하고 확장성을 보장합니다.
*   **API Gateway:** 외부 데이터(예: 쇼핑몰 API, 재무 API) 접근은 **Agent 2 (데이터 수집/통합)**를 통해 관리하며, 키 관리는 **Developer 설정**에 따라 환경 변수로만 접근하도록 엄격히 제한합니다.
