**✅ 시스템 작동 원리 (How It Works):**

저희의 시스템은 중앙 제어 장치인 **[Orchestrator]**를 중심으로 작동합니다. 마치 지휘자(Orchestrator)가 오케스트라의 모든 악기(에이전트 모듈)를 지휘하는 것과 같습니다.

1.  **입력 (Input):** 사용자가 목표(Goal)를 제시합니다.
2.  **지휘 (Orchestrator):** Orchestrator는 이 목표를 분석하여, 어떤 에이전트가, 어떤 순서로, 어떤 데이터를 가져와야 하는지 **데이터 흐름(Data Flow)**을 즉시 설계합니다.
3.  **실행 (API Call):** 각 에이전트 모듈은 독립적인 **API 엔드포인트**를 통해 호출됩니다. 예를 들어, '트렌드 분석'이 필요하면, YouTube Agent가 외부 데이터 소스에 API를 호출하여 최신 데이터를 수집합니다.
4.  **출력 (Output):** 수집된 데이터는 Designer Agent에게 넘어가 '브랜드 가이드라인'이라는 필터를 거쳐 비주얼로 구현되고, 최종적으로 Secretary가 이를 정리하여 사용자에게 전달됩니다.

**핵심은 이것입니다:** 모든 에이전트는 각자의 전문 분야를 가지고 독립적으로 작동하는 **'소프트웨어 모듈'**이며, 이 모듈들이 **지연 없이(Zero Latency)** 유기적으로 연결되어 하나의 목표를 향해 움직입니다.
