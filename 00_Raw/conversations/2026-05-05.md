## 1. 시스템 아키텍처 설계: 데이터 파이프라인 구조

JAY CORP 시스템은 **'시장 분석 → 운영 최적화 → 수익화 전략'**의 흐름을 자동화하는 것을 목표로 합니다. 따라서 데이터 파이프라인은 **중앙 집중식 데이터 허브(Data Hub)**를 중심으로 각 에이전트가 독립적으로 작업하되, 결과는 상호 연관되도록 설계합니다.

### 1.1. 아키텍처 개요 (Conceptual Flow)

| 단계 | 구성 요소 | 역할 | 주요 데이터 흐름 |
| :--- | :--- | :--- | :--- |
| **Input Layer (수집)** | 외부 API, 웹 크롤러, 내부 DB | 시장 데이터, SCM 데이터, 재무 데이터 등 원천 데이터 수집. | Raw Data $\rightarrow$ Data Ingestion |
| **Processing Layer (처리)** | 분석 에이전트 (Agent 1, 3, 4 등) | 수집된 데이터를 정제, 분석, 예측, 패턴 식별. | Raw Data $\rightarrow$ Processed Data |
| **Decision Layer (결정)** | 전략 에이전트, 재무 에이전트 | 처리된 데이터를 기반으로 최적의 전략(가격, 수량, 마케팅) 결정. | Processed Data $\rightarrow$ Strategy Output |
| **Output Layer (산출)** | Secretary, 최종 보고서 생성 모듈 | 분석 결과, 실행 가능한 스크립트, 보고서 형태로 최종 결과 제공. | Strategy Output $\rightarrow$ Final Report/Action |

### 1.2. 데이터 흐름 상세 (Data Flow Detail)

1.  **Data Ingestion (수집):**
    *   **소스:** 외부 시장 데이터 (크롤링), 내부 운영 데이터 (SCM), 재무 데이터 (매출/비용).
    *   **통합:** 모든 데이터는 **Central Data Store (CDS)**에 표준화된 JSON/CSV 형태로 저장됩니다.

2.  **Transformation & Analysis (변환 및 분석):**
    *   각 에이전트는 CDS에서 필요한 데이터 셋을 추출하여 자체적인 분석 로직(AI/ML)을 적용합니다.
    *   예: Agent 1 (크롤링) $\rightarrow$ Agent 4 (트렌드 분석) $\rightarrow$ Agent 3 (수량 예측).

3.  **Feedback Loop (피드백 루프):**
    *   **핵심:** 분석 결과는 반드시 **재무/KPI 에이전트**로 피드백되어, 모든 전략적 결정이 연매출 목표(100억)에 미치는 영향을 실시간으로 검증합니다.
    *   **결과:** 전략 에이전트가 제시한 결과는 다시 운영 에이전트의 실행 계획으로 연결되어, 실제 시스템에 적용됩니다.

### 1.3. API 및 연동 전략

*   **표준화된 인터페이스:** 모든 에이전트는 내부적으로 **JSON 기반의 표준화된 입출력 포맷**을 사용합니다. 이는 에이전트 간의 데이터 교환 오류를 최소화하고 확장성을 보장합니다.
*   **API Gateway:** 외부 데이터(예: 쇼핑몰 API, 재무 API) 접근은 **Agent 2 (데이터 수집/통합)**를 통해 관리하며, 키 관리는 **Developer 설정**에 따라 환경 변수로만 접근하도록 엄격히 제한합니다.

2) 재고 리스크: 대량 발주 → 시즌 종료 후 50%+ 재고 손실
3) 인건비/조직비: 기획자, 디자이너, MD, 생산팀, 마케터 등 인력 의존

AI 멀티모달 모델의 성숙(2024~2026)으로, 이 다섯 단계 모두를 AI Agent가 실시간으로 수행할 수 있는 임계점에 도달했다.
지금이 기존 패션 브랜드의 구조를 1인 기업이 뒤집을 수 있는 유일한 타이밍이다.


## 3. 솔루션: 5 AI Agent Company 아키텍처

바이브패션은 5개의 독립적 AI Agent를 각각 하나의 "회사"처럼 운영한다.
각 Agent는 표준 인터페이스(JSON, PDF, API)로 통신하며, 1인 창업자는 오케스트레이터 역할만 수행한다.

목표 사이클: 3~6개월 → 2~5일
핵심 가치: 속도(Speed), 정확도(Accuracy), 재고 최소화(Zero-Inventory)
전략: 소량 생산 → 시장 반응 검증 → 데이터 기반 스케일업
초기 카테고리: 스웨터, 팬츠 (좁은 SKU에서 시작 → 확장)


### Agent 1. 기획 Co. (상품기획 자동화)

INPUT
 - SNS 데이터: 인스타그램, 틱톡 (해시태그/계정 단위 크롤링)
 - 패션 플랫폼: 무신사, 지그재그 (베스트/신상/카테고리)

PROCESS
 - 멀티모달 AI 속성 태깅 (자체 택소노미 기준)
 - 시계열 트렌드 분석 + 트렌드 스코어링
 - 카테고리/타겟/예상 판매량/키 디자인 요소 추출

OUTPUT
 - 상품기획서 (구조화된 JSON + PDF)

운영 원칙
 - 주간 크롤링 사이클
 - 초기 넓게 수집 → 유효 채널만 프루닝하여 비용 최적화


### Agent 2. 디자인 Co. (디자인 자동화)

INPUT
 - 상품기획서 + 트렌드 레퍼런스 이미지

PROCESS
 - PIcasso.AI 기반 개인 시스템 재구축
 - 레퍼런스 → 디자인 변형 → 작업지시서 자동 생성
 - 수만 장의 작업지시서 학습 데이터 활용
 - 유사 작업지시서 검색용 Vector DB 구축

OUTPUT
 - 작업지시서 (스케치, 컬러웨이, BOM)

핵심 자산
 - 작업지시서 이미지 학습 데이터셋
 - Vector DB 기반 유사 디자인 검색 엔진


### Agent 3. 생산 Co. (생산 자동화)

INPUT
 - 작업지시서 (BOM 포함)

PROCESS
 - BOM 기반 원가 계산
 - 공장풀 매칭 알고리즘
 - 발주/납기 자동 관리

OUTPUT
 - 발주서, 생산 일정, 원가 리포트

수요예측 원칙
 - 최종 판매 소진율 90%를 목표로 초기 수량 예측
 - 트리거 기반 단순 리오더가 아닌, 데이터 기반 정밀 예측


### Agent 4. 영업 Co. (이커머스 + 물류)

INPUT
 - 완성 상품 정보 + 상품 이미지

PROCESS
 - AI 가상 모델 이미지 생성
 - 상세페이지 자동 생성
 - 멀티 플랫폼 동시 업로드

OUTPUT
 - 라이브 상품 페이지, 주문/물류 관리, 재고 모니터링

타겟 채널
 - 무신사, 지그재그, 자사몰


### Agent 5. 마케팅 Co. (마케팅 자동화)

INPUT
 - 상품 정보 + 실시간 판매 데이터

PROCESS
 - SNS 콘텐츠 자동 생성
 - AI 인플루언서 운영
 - ROAS 기반 A/B 테스팅

OUTPUT
 - 마케팅 콘텐츠, 예산 배분 리포트, 성과 대시보드


### 피드백 루프 (Closed-Loop Learning)

 - 영업 → 기획: 판매 데이터로 다음 시즌 기획 보정
 - 마케팅 → 기획: 마케팅 성과로 트렌드 스코어 재학습
 - 영업 → 생산: 재고 상태 기반 추가 생산 자동 판단


## 4. 기술 스택 (MVP)

언어/프레임워크
 - Python 단일 스택 (FastAPI, Celery, httpx, Playwright)

데이터/메시징
 - Redis: 에이전트 간 비동기 통신 메시지 브로커 + 캐싱
 - PostgreSQL: 상품/트렌드 데이터
 - MinIO (로컬 S3 호환): 이미지/문서 스토리지

AI 모델
 - Claude API: 멀티모달 태깅, 기획서 생성, 디자인 변형, BOM 생성
 - FLUX Kontext: 제품 사진 → 도식화 변환
 - Firecrawl MCP: 크롤링 인프라

운영 환경
 - Docker Compose: 로컬 개발
 - AWS (SQS, S3, ECS): 프로덕션 타겟

설계 원칙
 - 에이전트별 마이크로서비스 (FastAPI + pipeline.py + 이벤트 브로커)
 - 동일 패턴 반복으로 새 에이전트 추가 비용 최소화


## 5. MVP 범위 및 진행 상황

MVP 범위: Agent 1(기획 Co.) + Agent 2(디자인 Co.) E2E 파이프라인

이미 완료

 - Agent Architecture PPT (VibeFashion_AgentArchitecture.pptx)
 - 기획 Co. 크롤링(무신사) → Claude 멀티모달 태깅 → 트렌드 스코어링 → 기획서 자동 생성
 - 디자인 Co. 기획서 수신 → 레퍼런스 수집 → AI 디자인 변형 → 작업지시서 출력

## 6. 1인 기업 운영 모델

조직 구조
 - 인간: 1명 (전략, 의사결정, AI Agent 오케스트레이션)
 - AI Agent: 5개 회사 단위로 가상 운영
 - 외부 협력: 중국 ODM 공장 (운영 디테일, 메인 전략 아님)

인간이 직접 하는 일
 - 브랜드 방향성 결정
 - 에이전트 출력물 최종 승인 (특히 디자인, 마케팅 톤앤매너)
 - 신규 카테고리/시장 진입 의사결정
 - 에이전트 시스템 개선 및 학습 데이터 큐레이션

자동화되는 일
 - 트렌드 수집/분석
 - 디자인 생성 및 작업지시서 작성
 - 원가 계산 및 발주
 - 상세페이지/마케팅 콘텐츠 생성
 - 채널 업로드 및 재고 모니터링


## 7. 경쟁 우위 (Moat)

1) 도메인 전문성: 대기업 이커머스/IT 운영 경험 기반의 시스템 설계 깊이
2) 프로세스 자산: 수만 장의 작업지시서 학습 데이터 + 자체 택소노미
3) 통합 아키텍처: 5개 Agent가 표준 인터페이스로 연결된 폐쇄 루프
4) 비용 구조: 인력 비용 거의 없음 → 초저가 가격 정책 또는 초고마진 가능
5) 학습 효과: 매 시즌 데이터가 다음 시즌 정확도를 비선형적으로 향상


## 8. 리스크 및 대응

기술 리스크
 - AI 모델 품질 한계 → 인간 최종 승인 단계 유지
 - 크롤링 정책 변경 → 멀티 채널 분산, MCP 기반 추상화

시장 리스크
 - 초기 브랜드 인지도 0 → 좁은 SKU + AI 마케팅으로 회전율로 승부
 - 패션 트렌드 빗나감 → 소량 생산 + 빠른 사이클로 손실 최소화

운영 리스크
 - 1인 의존 → 시스템 자체가 회사이므로 인적 리스크 낮음
 - 공장 의존 → 중국 ODM은 초기 단계, 점진적으로 공장풀 확장


## 9. 핵심 메시지

바이브패션은 "AI로 패션 브랜드를 만든다"가 아니라,
"5개의 AI 회사가 협업하는 가상 기업 구조"를 만드는 프로젝트다.

이 구조는 패션을 시작점으로 하지만,
같은 아키텍처를 다른 소비재 카테고리로 확장 가능한 범용 1인 기업 운영 시스템이다.

목표는 단순히 의류를 파는 것이 아니라,
"한 사람이 AI Agent 회사들을 운영하여 시장에 빠르게 응답하는 차세대 기업 형태"를 증명하는 것이다.
